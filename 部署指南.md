# 英语学习平台 - 部署指南

## 📋 前置准备

### 1. 检查环境变量配置
确保项目根目录的 `.env` 文件配置正确：

```env
# 数据库
DATABASE_URL="file:./dev.db"

# JWT 密钥（生产环境请更换为随机字符串）
JWT_SECRET="your-secret-key-change-in-production"

# API 地址（可选）
NEXT_PUBLIC_API_URL="http://localhost:3000"
```

## 🚀 方案1：本地局域网访问（最简单）

### 适用场景
- 让同一局域网内的设备访问（如家里、办公室）
- 快速测试，无需配置服务器

### 操作步骤

1. **查看本机IP地址**
```bash
ipconfig
```
找到 "IPv4 地址"，例如：`192.168.1.100`

2. **修改 Next.js 配置**
编辑 `next.config.js`：
```javascript
/** @type {import('next').NextConfig} */
const nextConfig = {
  // ... 其他配置
}

module.exports = nextConfig
```

3. **启动开发服务器**
```bash
npm run dev
```

4. **局域网内其他设备访问**
在浏览器输入：`http://192.168.1.100:3000`

**注意**：需要确保防火墙允许3000端口访问

## 🌐 方案2：使用 ngrok 内网穿透（临时测试）

### 适用场景
- 快速分享给外网用户测试
- 临时演示，不需要长期使用
- 免费版有限制但足够测试

### 操作步骤

1. **安装 ngrok**
- 访问 https://ngrok.com/ 注册账号
- 下载 Windows 版本
- 解压到任意目录，例如 `C:\ngrok`

2. **配置 authtoken**
```bash
cd C:\ngrok
ngrok config add-authtoken 你的token
```

3. **启动穿透**
```bash
# 先启动你的网站
npm run dev

# 在另一个终端启动 ngrok
ngrok http 3000
```

4. **获取公网地址**
ngrok 会显示：
```
Forwarding  https://xxxx-xx-xx-xx-xx.ngrok-free.app -> http://localhost:3000
```

使用这个 https 地址就可以让任何人访问！

**注意**：
- 免费版每次重启地址会变化
- 有流量限制
- 适合临时测试，不适合生产环境

## ☁️ 方案3：部署到 Vercel（推荐，免费）

### 适用场景
- 正式上线运营
- 全球 CDN 加速
- 自动 HTTPS
- 完全免费（个人项目）

### 操作步骤

1. **安装 Vercel CLI**
```bash
npm install -g vercel
```

2. **准备生产环境数据库**

由于 Vercel 是 serverless 环境，需要使用云数据库：

**选项A：使用 Supabase（推荐，免费）**
- 访问 https://supabase.com/
- 创建新项目
- 获取数据库连接字符串
- 修改项目使用 PostgreSQL 而不是 SQLite

**选项B：使用 PlanetScale（免费MySQL）**
- 访问 https://planetscale.com/
- 创建数据库
- 获取连接字符串

**选项C：使用 Vercel Postgres（最简单）**
- Vercel 内置的 Postgres 服务
- 在 Vercel 项目设置中直接添加

3. **修改代码使用环境变量**

更新 `prisma/schema.prisma`：
```prisma
datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}
```

4. **部署到 Vercel**
```bash
# 在项目根目录
vercel

# 按提示操作：
# - 登录 Vercel 账号
# - 设置项目名称
# - 配置环境变量
# - 确认部署
```

5. **配置环境变量**
在 Vercel Dashboard 设置：
```
DATABASE_URL = 你的数据库连接字符串
JWT_SECRET = 生产环境密钥（随机字符串）
NEXT_PUBLIC_API_URL = 你的vercel域名
```

6. **运行数据库迁移**
```bash
# 设置生产环境数据库
vercel env pull .env.production

# 生成 Prisma Client
npx prisma generate

# 推送数据库 schema
npx prisma db push
```

**优点**：
✅ 完全免费（个人项目）
✅ 全球 CDN 加速
✅ 自动 HTTPS 证书
✅ 自动部署
✅ 无限带宽

## 🖥️ 方案4：购买云服务器部署（完全控制）

### 适用场景
- 需要完全控制服务器
- 数据敏感，需要自己管理
- 需要运行其他服务

### 推荐云服务商
- **阿里云**：https://www.aliyun.com/
- **腾讯云**：https://cloud.tencent.com/
- **华为云**：https://www.huaweicloud.com/

### 服务器配置建议
- CPU：2核及以上
- 内存：2GB 及以上
- 系统：Ubuntu 20.04 或 22.04
- 带宽：1Mbps 及以上

### 操作步骤

1. **购买服务器**
- 选择配置
- 设置 root 密码
- 获取服务器公网IP

2. **连接服务器**
```bash
ssh root@你的服务器IP
```

3. **安装 Node.js**
```bash
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt-get install -y nodejs
```

4. **安装 PM2（进程管理器）**
```bash
sudo npm install -g pm2
```

5. **上传项目代码**
```bash
# 在本地打包项目（排除 node_modules）
tar -czf english-learning.tar.gz --exclude=node_modules --exclude=.next .

# 在本地执行，上传到服务器
scp english-learning.tar.gz root@服务器IP:/root/

# 在服务器上解压
ssh root@服务器IP
cd /root
tar -xzf english-learning.tar.gz
```

6. **安装依赖**
```bash
cd /root/english-learning-platform
npm install
```

7. **配置环境变量**
```bash
nano .env
```
添加：
```env
DATABASE_URL="file:./prod.db"
JWT_SECRET="你的生产密钥"
NODE_ENV="production"
```

8. **构建项目**
```bash
npm run build
```

9. **使用 PM2 启动**
```bash
pm2 start npm --name "english-learning" -- start
pm2 save
pm2 startup
```

10. **配置 Nginx 反向代理**

安装 Nginx：
```bash
sudo apt update
sudo apt install nginx
```

配置 Nginx：
```bash
sudo nano /etc/nginx/sites-available/english-learning
```

添加配置：
```nginx
server {
    listen 80;
    server_name 你的域名 或 IP;

    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}
```

启用配置：
```bash
sudo ln -s /etc/nginx/sites-available/english-learning /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl restart nginx
```

11. **配置域名（可选）**
- 购买域名
- 在域名 DNS 设置中添加 A 记录指向服务器IP
- 配置 HTTPS（使用 Let's Encrypt 免费证书）

## 🔐 安全建议

### 1. 修改默认JWT密钥
```bash
# 生成随机密钥
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
```

### 2. 配置防火墙
```bash
# 只开放必要端口
sudo ufw allow 22    # SSH
sudo ufw allow 80    # HTTP
sudo ufw allow 443   # HTTPS
sudo ufw enable
```

### 3. 定期备份数据库
```bash
# 备份数据库
cp dev.db dev.db.backup.$(date +%Y%m%d)
```

### 4. 设置自动备份（crontab）
```bash
crontab -e
# 添加：每天凌晨2点备份
0 2 * * * cp /root/english-learning-platform/dev.db /root/backups/dev.db.$(date +\%Y\%m\%d)
```

## 🆘 常见问题

### Q1: 端口被占用怎么办？
```bash
# 查找占用进程
netstat -ano | findstr :3000
# 结束进程
taskkill /PID 进程ID /F
```

### Q2: 数据库迁移失败？
```bash
# 重新生成 Prisma Client
npx prisma generate
# 强制推送
npx prisma db push --force-reset
```

### Q3: Vercel 部署后数据库连接失败？
- 检查环境变量是否正确设置
- 确保 Prisma schema provider 为 postgresql
- 运行 `npx prisma db push`

### Q4: 域名如何配置？
- 在域名服务商添加 A 记录指向服务器IP
- 等待 DNS 生效（通常10分钟-24小时）
- 配置 Nginx server_name 为你的域名
