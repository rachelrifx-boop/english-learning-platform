# 视频自动处理功能说明

## 功能概述

系统现在支持对上传的视频进行自动化处理：

### ✅ 已实现的自动化功能

1. **自动翻译视频标题**
   - 使用 MyMemory Translation API（免费）
   - 自动将英文标题翻译为中文
   - 带备用翻译，防止 API 失败

2. **智能分析难度等级**
   - 基于字幕内容自动分析
   - 符合 CEFR 标准（A1-C2）
   - 考虑因素：
     * 平均词长
     * 句子长度
     * 高级词汇比例
     * 词汇多样性

3. **自动翻译字幕**
   - 批量翻译英文字幕
   - 生成中英双语字幕

---

## 使用方法

### 方法 1：上传视频时自动处理（推荐）

上传视频后，前端可以调用以下 API 自动处理：

```typescript
// 上传视频成功后调用
fetch('/api/admin/auto-process', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ videoId: 'your-video-id' })
})
```

### 方法 2：批量处理现有视频

运行命令处理所有需要处理的视频：

```bash
npm run auto-process
```

该脚本会自动：
1. 查找所有没有翻译或难度等级的视频
2. 批量翻译标题
3. 分析难度等级
4. 更新数据库

### 方法 3：单独处理某个功能

```bash
# 只翻译字幕
npm run add-translations

# 只生成视频封面
npm run create-covers
```

## 难度等级分析算法

### CEFR 等级标准

| 等级 | 难度得分 | 特征 |
|------|----------|------|
| A1 | < 1.5 | 基础词汇，短句子，词长 3-4 |
| A2 | 1.5-2.5 | 简单句，常用词汇，词长 4-5 |
| B1 | 2.5-4.0 | 中等词汇，复合句，词长 5-6 |
| B2 | 4.0-5.5 | 较复杂词汇，长句子，词长 6-7 |
| C1 | 5.5-7.0 | 高级词汇，复杂句式，词长 7+ |
| C2 | > 7.0 | 专业词汇，高度复杂，词长 8+ |

### 分析因素

1. **平均词长**（权重 25%）
   - 基准：3 字符
   - 每增加 1 字符 = 0.2 分

2. **句子平均长度**（权重 25%）
   - 每词贡献 0.05 分
   - 反映句子复杂度

3. **高级词汇比例**（权重 25%）
   - 词长 > 7 的词视为高级词
   - 比例越高，分数越高

4. **词汇多样性**（权重 12.5%）
   - Type-Token Ratio
   - 0.5 为基准

## API 限制

### MyMemory Translation API
- **免费配额**：每天 5000 次
- **速率限制**：有，建议每秒不超过 1 次请求
- **处理**：脚本已内置重试和延迟机制

### 建议策略
1. 批量处理时每视频间隔 1-2 秒
2. 优先处理上传时间较新的视频
3. 可以设置备用翻译映射

## 配置选项

### 自定义备用翻译

编辑 `scripts/auto-process-video.js` 或 `app/api/admin/auto-process/route.ts`：

```javascript
const fallbackTranslations = {
  'Your Video Title': '你的视频标题翻译',
  // 添加更多...
}
```

### 调整难度算法

修改 `analyzeDifficulty` 函数中的权重：
- `avgWordLength` - 词长权重
- `avgSentenceLength` - 句长权重
- `advancedRatio` - 高级词汇权重
- `diversity` - 词汇多样性权重

## 注意事项

1. **字幕要求**：难度分析需要英文字幕内容
2. **API 限制**：大量处理时注意速率限制
3. **准确性**：难度分析为参考值，管理员可手动调整
4. **错误处理**：API 失败时使用备用方案，不会中断流程
